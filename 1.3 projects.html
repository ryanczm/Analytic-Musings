Books/stuff to learn:

Last update - 18th Nov 2021 - This is a condensed version of my learning plan here. Structure a pseudo-syllabus of books per topic, then allocate partitions of ~7 days to each. The goal is to rotate through all topics for one cycle, and repeat cycles. To sample from each topic and chug along at a steady pace. I bookmark my progress here. Having finished my intern, I have been busy migrating to Notion, restructuring notes and setting up a PKM system, and am kicking myself for not doing it earlier because it is really good and I highly recommend it

  • Computer Architecture books - Tannenbaum → Ledin → Hennessy. Prereq is intro C/C++/Java, so I will do that first. Scrapped EOCS in favour of these.
  • Computer Networking books - Tannenbaum, Kurose.
  • Software design/best practices books - Abelson (SICP), Ousterhout, Martin, Hunt, Robson. Prereq is intro C/C++/Java, so in order to understand these, I have to learn that first.
  • Theoretical CS books - Goldwasser, Bhargava, Cormen (DS&A) - most important, Moore, Hopcroft, Sipser, Arora.
  • Linear Algebra/Calculus - Linear Algebra: Axler. Calculus: Thomas, Riley, Miklavcic, Hubbard. The latter is the true mathematics style book, the former are more applied.
  • Graph Theory - Gould. Not that important for now, links to CS DS&A.
  • Mathematical Proofs - Chartrand. Soft prereq to Analysis.
  • Differential Equations - To read Zill and Simmons.
  • Analysis - Abbott, Tao I/II, Axler (MIRA) - The main roadblock for now and the gateway to intermediate mathematics. Goal is to gain enough mathematical maturity to learn introductionry Functional (Kreyszig) and Complex analysis (Churchill). Then I can branch out into other areas like abstract algebra, measure theory and DEs proper. Going to be a long and painful ride.
  • Statistics - Probability and statistics: Blitzstein, Wackerly, Krishnamoorthy. Linear models: Agresti.
  • Bayesian Statistics - Theory : Gelman, McElreath. Code: Cameron, Martin, HSE Coursera. Need to check math prereqs.
  • ML stuff - Time series books: Code - Brownlee → Lazzeri → Korstanje → Cotton. Theory - Montgomery → Ender → Stoffer → Durbin → Tsay. Should try to explore the various packages and ecosystems.
  • ML stuff - NLP courses - Andrew Ng Deeplearning.ai NLP specialization (Coursera), books - Antic, Loper (NTLK), Altinok (spaCy), Jurafsky (bible), Ravichandran (transformers), Rothman (transformers). I've only started on Jurafsky so far.
  • Classical ML stuff - Selected chapters from ESL, Bishop and Murphy. For DL, Bengio, Zhang (for code) for books, and MIT 6.S191 for courses (intuition), and PyTorch official tutorials.
  • DE stuff- Data engineering. Kleppmann, Weber, then books on Flask (Grinberg), AWS (Artasanchez), Docker (Cook), Airflow (Ruiter), Kafka (Bejeck), Spark (Damji) Prereqs is Bash and any app-development framework (e.g Node). Go through those first.
  • Frontend - HTML/CSS: Robson and Frain. JS: Simpson → Flanagan, Wieruch (React), Wexler → Casciaro (Node). More out of curiousity and exposure.
  • Backend - Not so much to actually use them, but as prereqs to the CA books. C (Seacord, Prata), Java (Schildt, Horstmann), C++ (Prata, Lippmann). It's not that I want to learn these languages to use them, but rather just so I can understand the above CS books.
  • Science stuff - Have put together some recos from my NUS Physics friend (thanks McCoy!) (who told me Susan Fowler's syllabus is actually pretty bad) and MIT OCW respectively. The more you know ...
  • Econs stuff - I'm trying to find a good monetary economics book with minimum math, or at least light on math. If anyone knows please let me know.
  • Cognitive framework or "Theory of Learning"- to expound on this framework, document it and test it by self-learning.
