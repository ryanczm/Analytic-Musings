Books/stuff to learn:

Last update - 11th July 2021 - This is a list of stuff to learn, for the long term, for bookkeeping. The plan is to structure a pseudo-syllabus consisting of textbooks for each topic, then allocate time (blocks of ~5 days per topic) Within each topic are the subtopics consisting of books. The goal is to spread time evenly and not overly focus on one topic. The strategy is not completion (which is impossible), but to sample from each topic and chug along at a steady pace, switching topics once the window is up, ad infinitum. This post is just to keep track, and should keep me busy for a long time.

  • Computer Architecture books - Tannenbaum → Ledin → Hennessy. Prerequisite seems to be introductory low-level language (e.g C), so I will do that first. Scrapped EOCS in favour of these.
  • Computer Networking books - Tannenbaum, Kurose.
  • Software design/best practices books - Abelson (SICP), Ousterhout, Martin, Hunt, Robson. The prerequisites seem to be introductory C and Java, so in order to understand these, I have to learn that first.
  • Theoretical CS books - CLRS, Moore, Hopcroft, Sipser, Arora.
  • Linear Algebra/Calculus - To attempt Axler's Linear Algebra Done Right and Hubbard's Vector Calculus(I've only attempted the start of both of these so far). This would be the foundation for the ML/DL books below.
  • Graph Theory - Gould
  • Mathematical Proofs - Chartrand. I have always struggled with proofs, so I need to try and look at this.
  • Differential Equations - To read Zill and Simon's.
  • Analysis - Tao's Analysis I/II, Axler's MIRA - The main roadblock for now and the gateway to intermediate mathematics. Anyway, the goal is to gain enough mathematical maturity to learn introductionry Functional (Kreyszig) and Complex analysis (Churchill). Once I do that I will go back to basics and attempt some proofs from the beginning, since I have only a surface level understanding of proofs (i.e never tried to prove things myself). Don't see myself getting to this point for a long while.
  • Statistics - Probability and statistics: Blitzstein's Introduction to Probability, Wackerly's Mathematical Statistics with Applications, Krishnamoorthy's Statistical Distributions, for regression models: Draper's Applied Regression Analysis, for GLMs: Agresti's Foundations of Linear and Generalized Linear Models.
  • Bayesian Statistics - Gelman,McElreath, Cameron and Martin.
  • ML stuff - Time series books (Lazzeri, Brownlee, Korstanje, Stoffer/Montgomery, Koopman). Once I have done that, I can read Tsay. However this is going to take a very long while to make progress in at all.
  • ML stuff - NLP books - Loper (NTLK book), Lane (NLPIA), Jurafsky (bible), Ravichandran (transformers), Rothman (transformers). I've only started on Jurafsky so far.
  • Classical ML stuff - Selected chapters from ESL, Bishop and Murphy. For DL, Bengio, Zhang. However for Bengio a sturdy foundation in linear algebra and vector Calculus, background in information theory and numerical analysis is required. So this means I have to properly attempt Axler and Hubbard, which I have not yet done so due to time constraints. Of course, I could just go ahead and focus on the coding aspect instead of the theory aspect.
  • DE stuff - Data engineering. I've managed to structure an order of learning and found some books/courses, starting with the general Kleppmann's Designing Data Intensive Applications, Weber's Data Science in Production, then books on Flask (Grinberg), AWS (Wittig), Docker (Cook), Airflow (Ruiter), Kafka (Bejeck), Spark (Damji) respectively. Edit: Unfortunately, I have realized Bash scripting (Shotts) is a necessary pre-req, so I will do this one first.
  • Frontend - JavaScript and JavaScript frameworks/libraries. JS (Simpson), JS (Flanagan), Wieruch (React), Casciaro (Node), Freeman (Angular) I've managed to scrape together some books, but this is a long way down the road. This list is very rudimentary and I feel like it's missing stuff. I also feel like this is one of the areas where doing > learning, but still. Books are a static source of learning and perhaps YouTube videos/doing stuff will work better.
  • Backend - Some introduction to lower level languages, are prerequisites for the above CS books which I really want to read. C (Seacord, Prata), Java (Schildt, Horstmann), C++ (Prata, Lippmann). It's not that I want to learn these languages to use them, but rather just so I can understand the above CS books.
  • Science stuff - I've put together the structure/syllabus of sorts of textbooks each for undergraduate Physics, Chemistry, and Biology from Susan Fowler's website and MIT OCW respectively, but I mostly plan to focus on Physics and Biology.
  • Econs stuff - I'm trying to find a good monetary economics book with minimum math, or at least light on math. If anyone knows please let me know.
  • Cognitive framework or "Theory of Learning"- to expound on this framework, document it and test it by self-learning.
