Introduction

Note/Disclaimer - 10th April 2021: As of quite a while already (past 5 months), I have stopped using this post as a reference of stuff I need to look at, having been replaced by the list on the Projects page. This originally was made when I wanted to start my self-learning journey in Dec 2020, and is remarkably amateurish, disorganized, and shallow (despite being rather organized). I will probably try to update this page to be in sync with the official one I use at Projects page.

This is a list of useful resources I've gathered for self-learning, and I wanted to make a central reference for myself. Unfortunately, my degree does not cover much of the fields I’d like to learn, and thus the onus is to figure out, how best to self-learn these topics. So, I set out to find resources online that I could use, related to finance, data science, and math (in general) and categorized them like so. Edit: As mentioned above, this list is outdated for everything but the finance section, so here is the updated one.

Key Themes

  1. Roadmaps for technical subjects are important, as you need to build up from the basics.
  2. The challenge is to find not just the resources themselves, but a guide that tells you in which order to learn concepts. This is important if the field is in-depth and concepts are built upon a lot of foundational concepts.
  3. Different fields require different ways of thinking, or “metacognition”- being aware of your thought process, so the challenge is to configure your thought process to make it optimal for that particular subject, and be able to “switch” between modes of thought processes.
  4. Doing this will take a lot of time.

Programming:

Python

  1. Corey Schafer's playlist is quite comprehensive for an introduction. There are 143 videos. I've only done like a third so far.
  2. Derek Banas’ YouTube channel for the common visualization libraries and packages (Numpy, Pandas, MPL, SNS, Plotly, etc)
  3. Books: I feel like it’s quite easy to find books on Python, so I didn’t bother searching for any.

R:

  1. Books on R by Garrett Grolemond and Hadley Wickham (Tidyverse creator) and Robert Kabacoff. I feel like books are better to learn for R, for some reason.

CS Books:

TO READ- IMPORTANT

  1. Elements of Computing Systems (nand2Tetris) - Noam Nisham. Bottom-up from hardware to abstraction of a computer.
  2. Computer Networks- A Top-Down Approach Kurose and Ross. Computer Networks and the Internet

  4. Computer Systems Bryant and O'Halloran. The alternative of EOCS.

Books:

Code-based

  1. Hands-On Machine Learning (Aurelion Geron)
  2. Deep Learning (apparently a famous DL book by Bengio and Goodfellow, but probably way too advanced, but nonetheless)
  3. Approaching almost any Machine Learning Problem (Abhishek Thakur)
  4. Speech and Language Processing: An Introduction to Natural Language Processing NLP (Jurafsky Daniel)

Theory-based

  1. Machine Learning- A Probabalistic Perspective (Kevin Murphy). Bit too hard now, covers everything.
  2. Introduction to/Elements of Statistical Learning (Hastie & Tibshirani). ISL and ESL.
  3. Pattern Recognition and Machine Learning. Bishop's famous PRML. Apparently easier than Murphy.
  4. Deep Learning by Bengio, Goodfellow, and Courville. The comprehensive DL book. Will I ever get to this point? Hopefully..

Forecasting

  1. Forecasting: Principles & Practice (Hyndman George). Written in R and apparently simple, so maybe not.
  2. Time Series by Shumway and Stoffer. Apparently easier than Hamilton and so more accessible for this topic.
  3. Time Series by Hamilton Apparently the bible for TS, but not accessible.
  4. Intro to Time Series Forecasting with Python by Brownlee. Just a simpler, code-oriented book to teach simple forecasting in Python.
  5. I know TS is usually regarded as statistics, not ML per se.
